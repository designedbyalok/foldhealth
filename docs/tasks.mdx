The **Tasks** page (`#/tasks`) is where you track everything assigned to you, your task pool, items you created, and threads where you were mentioned. Switch between a grouped list view and a drag-and-drop kanban board.

## When to use it

- Triage incoming care-coordination tasks at the start of a shift.
- Move work between **Pending**, **Missed**, and **Completed** as it progresses.
- Sort or group your task pool by status, priority, or due date to find what to do next.

## Tabs

The top of the page has four tabs with live count badges:

- **Assigned to Me** — tasks where you are the assignee.
- **My Task Pool** — unassigned tasks in your group that you can claim.
- **Created by Me** — tasks you opened.
- **Mentions** — threads where you were @-mentioned in a comment.

## List view

The default list view groups tasks by status. Each row shows:

- A circular checkbox (green check when completed).
- Task name, with the parent task name when the row is a subtask.
- A meta line with the care journey, automation source, and author.
- Attachment and comment counts.
- Priority icon (high, medium, low).
- Status badge, due date, assignee, and labels.

## Kanban board

Switch to the board view to drag cards between **Pending**, **Missed**, and **Completed** columns. The change persists immediately. Single-clicking a card opens the preview drawer; dragging never triggers the click.

## Filter, group, and sort

The filter bar has nine chips:

| Chip            | Purpose                                                              |
|-----------------|----------------------------------------------------------------------|
| Assigned to     | Filter by assignee.                                                   |
| **View By**     | Group tasks by **Status** (default), **Priority**, or **Due Date**.   |
| **Sort By**     | Within each group, sort by **Due Date**, **Priority**, or **Name**.   |
| Created By      | Filter by task creator.                                               |
| Members         | Filter by group member.                                               |
| Task Status     | Restrict to a subset of statuses.                                     |
| Priority        | High, Medium, Low, None.                                              |
| Due Date        | Overdue, Today, Upcoming, No Due Date.                                |
| Labels          | One or more labels.                                                   |

Select **Clear All** to reset every chip.

## Edit a task inline

- **Status** — select the colored badge to switch between Pending, Missed, and Completed without opening the drawer.
- **Labels** — select an existing label chip to open the multi-select directly. The **Add Label** placeholder only appears when no labels are assigned.
- **Description** — open the task drawer and select the description area to edit rich text with the inline formatting toolbar (bold, italic, underline, strikethrough, list).

## Row actions

Hover any row (or the top-right corner of a kanban card) to reveal a three-dot menu with:

- **Mark as Complete / Missed / Pending** — depends on the current status.
- **Delete** — destructive, shown in red.

## Add a task

Select **+ Add Task** in the top-right of the page to open the create flow. Set the assignee, status, due date, priority, and labels before saving.
